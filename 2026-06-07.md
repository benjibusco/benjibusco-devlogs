Refactored deployment timestamp handling across the deployment infrastructure.

Separated:

```text

```

```text

```

Purpose:

- `updatedAt`

- `lastDeployedAt`

Updated `deploy-lib.sh` to preserve deployment timestamps during:

- no-change checks
- cron polling cycles
- deployment metadata rewrites

Deployment JSON files now maintain stable deployment history while cron continuously refreshes deployment health metadata.

This allowed safe staging API edits while preventing cron reset loops during active backend development.
