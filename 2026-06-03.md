### Deterministic Deployment Architecture

Refactored deployment workflows away from merge-based deployments:

```bash
git pull origin <branch>
```

Standardized deployments to use deterministic mirror-based syncing:

```bash
git fetch origin <branch>
git reset --hard origin/<branch>
```

New deployment flow:

```text
GitHub
→ GitHub Actions
→ VPS
→ VPS mirrors repository state exactly
```

Benefits:

- prevents deploy-time merge conflicts
- removes VPS repository drift
- improves staging/production consistency
- avoids local working tree conflicts
- better aligns with immutable deployment workflows
