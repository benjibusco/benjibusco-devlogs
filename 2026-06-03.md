## Deployment Architecture Refactor

Refactored deployment workflows away from:

```bash
git pull origin <branch>
```

and standardized deployments to use:

```bash
git fetch origin <branch>
git reset --hard origin/<branch>
```

This changed deployment behavior from merge-based pulls to deterministic mirror-based deployments.

New deployment model:

```text
GitHub
→ GitHub Actions
→ VPS
→ VPS mirrors repository state exactly
```

Benefits:

- prevents merge conflicts during deploys
- removes VPS drift issues
- ensures staging/production consistency
- avoids local working tree conflicts
- better aligns with immutable deployment workflows
