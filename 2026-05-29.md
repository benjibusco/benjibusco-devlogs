## Phase 7 — Browser Close / Tab Close Handling

Completed implementation of tab-scoped authentication behavior for protected lab pages.

Initial attempts included:

- `pagehide`
- `navigator.sendBeacon()`
- `beforeunload`
- `fetch(... keepalive)`

Testing showed browser-close requests were not consistently reaching the backend logout endpoint.

After evaluating browser behavior, shifted to a frontend tab-session architecture using:

```js
sessionStorage;
```

Implemented:

- `activateTabSession()`
- `clearTabSession()`
- `hasActiveTabSession()`

Authentication now requires:

```text
Valid backend session
+
Active browser tab session
```

Current behavior:

```text
Refresh page
→ Remains authenticated

Close tab/browser
→ Tab session destroyed

Open protected page again
→ Login required
```
