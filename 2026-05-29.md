## Phase 7 — Browser Close / Tab Close Handling

Implemented tab-scoped authentication behavior for protected lab pages using:

```js
sessionStorage;
```

Initial browser-close logout attempts included:

- `pagehide`
- `beforeunload`
- `navigator.sendBeacon()`
- `fetch(... keepalive)`

Testing showed browser-close requests were unreliable across browsers, so authentication was redesigned around frontend tab-session validation.

Implemented:

```js
activateTabSession();
clearTabSession();
hasActiveTabSession();
```

Authentication now requires:

```text
Valid backend session
+
Active browser tab session
```

Behavior:

```text
Refresh page
→ Remains authenticated

Close tab/browser
→ Tab session destroyed

Open protected page
→ Login required
```
