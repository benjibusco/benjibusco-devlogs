### Modal Refresh Persistence

Implemented persistent modal behavior during automatic deployment refresh cycles.

Previous behavior:

```text
auto refresh
→ deployment grid rerender
→ modal closes
→ user loses operational context
```

New architecture preserves modal state during deployment refreshes using centralized state tracking.

Added persistent state handling through:

```js
state.activeLogTarget;
state.activeLogMode;
state.activePm2Target;
```

Implemented refresh helpers:

```js
async function refreshOpenLogModal()
async function refreshOpenPm2Modal()
```

Current refresh lifecycle:

```text
loadDeployments()
→ render deployment cards
→ refresh open modal
→ update modal content in place
```

This now allows:

```text
open modal
→ auto refresh runs
→ deployment cards rerender
→ modal remains open
→ modal content refreshes automatically
```

The deployment dashboard now behaves more like a continuous operational monitoring interface instead of a static frontend panel.

## Deployment Timestamp Architecture

Resolved a major deployment state issue where deployment cards displayed stale deployment timestamps after successful cron deployment cycles.

Root issue:

```text
lastDeployedAt
```

only updated during successful deployments while:

```text
up-to-date
```

cron checks preserved outdated timestamps.

Implemented deployment timestamp preservation logic inside:

```text
deploy-lib.sh
```

Added helper systems to:

- preserve existing deployment timestamps
- detect reflog deployment timestamps
- maintain accurate deployment history
- avoid timestamp resets during status-only checks

The deployment dashboard now correctly separates:

```text
deployment history
```

from:

```text
cron refresh metadata
```

allowing deployment state to remain operationally accurate during continuous cron polling.
