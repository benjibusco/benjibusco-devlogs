## Deployment Monitoring System Refactor

### Environment-Aware API Architecture

Refactored the API monitoring system so both production and staging environments can share the same codebase without overwriting environment-specific paths or runtime configuration.

Implemented runtime environment detection using filesystem path checks such as:

```js
const isStaging = __dirname.includes("/api.dev.benjibusco.cloud/");
```

Used this pattern throughout the deployment system to dynamically configure:

- deployment metadata directories
- API roots
- PM2 process names
- deployment branch labels
- monitoring environment labels
- frontend API endpoints

This significantly improved long-term maintainability of the deployment architecture and reduced merge conflicts between staging and production branches.

### PM2 Runtime Monitoring

### Merge-Friendly Deployment Architecture

Refactored deployment scripts into a merge-safe environment-aware structure.

Updated:

- `server.js`
- `routes/deploy.js`
- `deploy-lib.sh`
- all deployment wrapper scripts

to support clean staging → main merge workflows.

Removed hardcoded deployment-lib sourcing such as:

```bash
source "/var/www/api.benjibusco.cloud/app/scripts/deployments/deploy-lib.sh"
```

and replaced it with portable runtime-relative sourcing:

```bash
source "$SCRIPT_DIR/deploy-lib.sh"
```

This allows:

- staging deployments to run independently
- production deployments to remain isolated
- identical deployment logic across environments
- future branch merges with minimal manual conflict resolution
