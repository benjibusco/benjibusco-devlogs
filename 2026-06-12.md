### Root Cause Discovery

Eventually isolated the primary authentication issue to special-character handling inside:

```text
DB_PASSWORD
```

Credentials worked manually through interactive:

```bash
psql
```

but inconsistent parsing across:

```text
dotenv
Node
pg
connect-pg-simple
PM2
```

caused runtime authentication failures.

Validated the fix through direct Node runtime pool testing:

```bash
node -e 'require("dotenv").config(); ...'
```

Confirmed successful runtime database connectivity after simplifying password formatting.

Established safer credential formatting guidance for future environment variables:

```text
letters
numbers
_
-
```

Avoiding special-character parsing edge cases across:

- dotenv
- bash
- PM2
- deployment scripts
- Node runtime parsing
