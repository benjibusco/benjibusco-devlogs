## Frontend Infrastructure

### Shared API Helper Architecture

Resolved a frontend infrastructure issue affecting authentication and protected deployment tooling after production deployment.

Issue:

```text
apiPost is not defined
```

Affected systems:

- login flow
- logout flow
- auth guards
- protected redirects
- deployment authentication checks

Root cause:

```text
scripts/core/api.js
```

contained reusable helper functions that were not reliably exposed globally across non-module browser scripts after the frontend refactor.

Stabilized the architecture through explicit browser exports:

```js
window.getApiBaseUrl = getApiBaseUrl;
window.getApiEnvironmentLabel = getApiEnvironmentLabel;
window.buildApiUrl = buildApiUrl;
window.apiRequest = apiRequest;
window.apiGet = apiGet;
window.apiPost = apiPost;
```

This restored reliable cross-script access for:

```text
auth.js
login.js
deployments.js
future reusable frontend systems
```

and aligned the frontend architecture with the current shared browser-script strategy.
