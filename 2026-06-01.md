## Dark Mode Feature Development

Implemented the first production-ready version of dark mode for the portfolio website using a centralized theme architecture.

### Theme Architecture Refactor

Refactored the CSS architecture to support theme switching through CSS custom properties.

Expanded `variables.css` to include:

- surface colors
- alternate surface colors
- typography variables
- muted text colors
- accent color variants
- border colors
- overlay colors
- shadow systems
- hero-specific variables
- transition variables

Created dedicated light and dark theme variable sets using:

```css
:root;
```

and

```css
[data-theme="dark"]
```

This established a scalable theming system that can be expanded later without additional large-scale refactors.

### CSS Conversion

Removed hardcoded colors throughout the project and migrated styling to theme variables.

Refactored:

- base.css
- components.css
- hero.css
- about.css
- services.css
- contact.css
- testimonials.css
- layout.css

Replaced hardcoded:

- white
- black
- rgba values
- static borders
- static shadows

with reusable variables such as:

```css
var(--color-surface)
var(--color-text)
var(--color-text-muted)
var(--color-border)
var(--shadow-soft)
var(--shadow-hover)
```

### Hero Dark Mode Support

Created dedicated hero theme variables:

```css
--color-hero-text
--color-hero-overlay-start
--color-hero-overlay-end
--shadow-text
```

Updated hero styling to use theme-aware overlays and typography.

Dark mode now applies a softer cinematic navy overlay while preserving image visibility and text contrast.

### Theme Toggle V1

Implemented the first working theme toggle system.

Features:

- toggle between light and dark mode
- updates `data-theme`
- stores user preference
- persists after page refresh
- persists after browser restart

Created:

```text

```
