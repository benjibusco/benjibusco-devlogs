## Freelance Assets Upload Architecture

Built the file upload system as runtime infrastructure separate from Git-tracked application code.

Runtime upload roots:

```text
/var/www/api.dev.benjibusco.cloud/uploads/
/var/www/api.benjibusco.cloud/uploads/
```

Active upload structure:

```text
uploads/
└── freelance/
    └── assets/
```

Confirmed that uploaded files should not live inside the repository or be pushed to GitHub. File storage is environment-specific, while application code remains merge-friendly between staging and production.

Configured upload behavior through environment variables:

```env
UPLOADS_ROOT=/var/www/api.dev.benjibusco.cloud/uploads
UPLOAD_MAX_FILE_SIZE_MB=25
```

Production uses the matching production upload root:

```env
UPLOADS_ROOT=/var/www/api.benjibusco.cloud/uploads
UPLOAD_MAX_FILE_SIZE_MB=25
```

### Upload Flow

Implemented the upload flow:

1. Validate the asset exists.
2. Accept a single `file` field through `FormData`.
3. Store the uploaded file under the configured upload root.
4. Update the existing asset metadata.
5. Return the updated asset record.

Updated asset metadata includes:

- `storage_path`
- `original_filename`
- `stored_filename`
- `mime_type`
- `size_bytes`
- `updated_at`

### Download Flow

Implemented the download flow:

1. Look up the asset record.
2. Confirm file metadata exists.
3. Resolve the relative path safely.
4. Confirm the physical file exists.
5. Send the file as an attachment using the original filename.

Downloads now go through the authenticated API route instead of exposing `/uploads` publicly.
