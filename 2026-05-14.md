# Devlog Automation System

Built an automated documentation pipeline using GitHub Actions.

Created `.github/workflows/update-devlog-index.yml` to:

- scan all `2026-*.md` files
- regenerate `devlogs.json`
- commit updated metadata automatically

Documentation flow:

```text
Write markdown
      ↓
Push to GitHub
      ↓
GitHub Actions regenerates devlogs.json
      ↓
Lab platform renders updated content
```

Added cross-repo automation so `ben-dev-log` triggers the lab repo automatically using `repository_dispatch`.

Created a second workflow in the lab repo to:

- update the `content/devlogs` submodule
- commit updated submodule pointers
- trigger deployment automatically

Updated the deploy workflow to run:

```bash
git submodule update --init --recursive
```

This allows all current and future submodules to deploy automatically without additional workflow changes.

Final deployment pipeline:

```text
Devlog markdown update
        ↓
GitHub Actions regenerates devlogs.json
        ↓
ben-dev-log triggers lab repo
        ↓
lab updates submodule pointer
        ↓
lab deploy workflow runs
        ↓
VPS pulls latest repo + submodules
        ↓
docs page updates automatically
```
